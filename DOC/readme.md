## 软件概述

Refiner 是一个基于 Cleanlab 的 YOLO 目标检测标签修正工具，通过 Web UI 自动定位可能有误的标注样本，支持人工审核和修正。

### 核心功能

- **自动问题检测**：使用 Cleanlab 分析 GT 标签和模型预测，识别三类问题：
  - **Overlooked（漏标）**：模型预测但 GT 未标注的目标
  - **Swapped（类别错误）**：GT 类别与预测类别不一致
  - **Bad Located（框不准）**：GT 框与预测框位置偏差大

- **直接标注模式**：可跳过 Cleanlab，不运行分析也能直接进入标注工具，对图片进行标注/修正（GT/Pred 可选加载）

- **交互式标注编辑**：提供完整的鼠标和键盘操作，支持框的创建、移动、调整大小、修改类别等

- **智能文件管理**：自动跳过已处理的样本，支持临时文件保存和确认机制

## Dashboard 页面操作

Dashboard 是分析配置和问题查看页面，分为三个区域：左侧配置面板、中间问题列表、右侧可视化面板。

### 1. 配置路径

在左侧配置面板中，需要设置以下路径：

#### 必需路径

- **Images Path**：图片目录路径
  - 示例：`/path/to/images`
  - 点击 `Parse Data` 后显示路径状态与图片数量

- **Output Path**：输出路径（**必填**）
  - 修正后的标注将保存到此路径
  - 默认值：`/home/yangxinyu/Test/Data/internalVideos_fireRelated_keyFrameAnnotations_verifying`
  - 系统仅确保 Output 根目录存在；子目录在保存时按需创建
  - 如果未设置，运行分析时会弹窗警告并拒绝执行

#### 可选路径

- **GT Labels Path**：GT（Ground Truth）标签目录路径（可选）
  - 示例：`/path/to/gt_labels`
  - 点击 `Parse Data` 后显示 `valid/missing_img`（排除 `*_tmp.txt`）

- **Pred Labels Path**：模型预测标签目录路径（可选）
  - 示例：`/path/to/pred_labels`
  - 点击 `Parse Data` 后显示 `valid/missing_img`

- **Human Verified Annotation Path**：人工验证路径（可选）
  - 已人工验证的标注路径
  - 如果设置，系统会跳过此路径中已存在的样本
  - 可以为空
  - 点击 `Parse Data` 后显示 `valid/missing_img`（仅统计 `{stem}.txt`）

- **Classes File**：类别映射文件（可选）
  - 用于显示类别名称的映射文件
  - 可以为空

- **Parse Data**：手动触发路径解析与统计（位于 `RUN ANALYSIS` 上方）

- **Pending Samples**：待处理样本数（醒目显示）
  - 定义：`Images ∩ (GT if set) ∩ (Pred if set) - Output - Human Verified`
  - 当 GT 或 Pred 未填写时，缺省侧视为全集

### 2. 运行分析

1. **点击 "Parse Data"**：解析路径并刷新 pending 与各路径统计
2. **设置 TopK 值**：在配置面板底部设置要显示的问题数量（默认 10）
3. **选择是否备份**：勾选 "Enable Backup" 可在分析前备份 GT Labels Path
4. **点击 "RUN ANALYSIS"**：
   - 仅当 **Images Path + GT Labels Path + Pred Labels Path** 均已设置且存在时才允许执行
   - 如果 Output Path 未设置，会弹窗警告并拒绝执行
   - 开始分析后，进度条会显示分析进度
   - 分析完成后，问题列表会显示三类问题

### 3. 查看问题列表

分析完成后，中间区域会显示三列问题列表：

- **Overlooked（橙色）**：漏标问题
- **Swapped（红色）**：类别错误问题
- **Bad Located（紫色）**：框位置不准问题

每列显示：
- 问题数量（顶部标签）
- 问题列表（按严重程度排序，分数越低越严重）
- 每个问题项显示：图片路径、问题框索引、严重程度分数

### 4. 可视化问题

- **点击问题项**：右侧可视化面板会显示该问题的可视化图片
  - 绿色实线框：GT 框
  - 蓝色虚线框：Pred 框
  - 红色高亮框：问题 GT 框（Swapped/Bad Located）
  - 橙色高亮框：问题 Pred 框（Overlooked）

### 5. 选择问题类型

在配置面板底部，有三个复选框：
- **Overlooked**：选择是否包含漏标问题
- **Swapped**：选择是否包含类别错误问题
- **Bad Located**：选择是否包含框位置不准问题

**重要**：
- 可以多选，进入标注工具时会包含所有选中类型的问题
- 当 **分析已完成** 时，必须至少选择一个类型，否则无法进入标注工具
- 同一图片可能出现在多种类型中，系统会自动去重（分析模式）

### 6. 进入标注工具

存在两种进入方式：

#### A. 分析模式（RUN ANALYSIS 之后）

1. 设置 TopK 值（如果需要）
2. 勾选需要标注的问题类型
3. 点击 **"Go to Annotation Tool"** 按钮
4. 系统会构建标注队列（只包含选中类型的问题，已去重）

#### B. 直接标注模式（跳过分析）

1. 至少设置 **Images Path** 与 **Output Path**
2. 可选设置：GT Labels Path / Pred Labels Path / Human Verified Annotation Path
3. 点击 **"Go to Annotation Tool"** 按钮
4. 系统会构建标注队列：
   - 样本集合：`Images ∩ (GT if set) ∩ (Pred if set)`
   - 自动排除：Output/Human Verified 中已存在 `.txt` 或 `*_tmp.txt` 的样本
5. 初始标注框：
   - GT：editable（可编辑）
   - Pred：reference（不可编辑）

## Annotator 页面操作

Annotator 是交互式标注编辑页面，分为四个区域：Viewer（主图像区）、Navigator（小地图）、Box List（框列表）、Control Panel（控制面板）。

### 1. Viewer（主图像区）

Viewer 是主要的图像显示和编辑区域。

#### 图像显示

- 图像会自动适应容器大小，保持宽高比
- 横向图片：填充宽度，垂直居中
- 纵向图片：填充高度，水平居中
- **自动聚焦**：由 `Auto Focus` 勾选控制。勾选时切图自动聚焦；取消勾选时切图保持 1x 显示

#### 鼠标操作

- **点击框**：选中框（显示 8 个控制点）
- **拖拽框**：移动框位置
- **拖拽控制点**：调整框大小
  - 4 个角点：同时调整宽度和高度
  - 4 个边点：只调整对应边的位置
- **空白区域拖拽**：创建新框（支持任意缩放级别，不再限制在1x）

#### 滚动条

- **垂直滚动条**：在放大状态下，上下滚动图像
- **水平滚动条**：在放大状态下，左右滚动图像

### 2. Navigator（小地图）

Navigator 显示整个图像的缩略图，用于快速导航。

- **点击 Navigator**：Viewer 会定位到点击位置（仅在放大状态下有效）
- **拖拽 Navigator**：连续移动 Viewer 的视野

### 3. Box List（框列表）

显示当前图片的所有框（GT + Pred）。

- **点击框项**：选中该框（仅可编辑框可选中）
- **点击眼睛图标**：切换框的显示/隐藏
- **颜色标识**：
  - 绿色：GT 框
  - 蓝色：Pred 框
  - 红色边框：当前选中的框
  - "(ref)" 标记：参考框（不可编辑）

### 4. Control Panel（控制面板）

#### Display Options（显示选项）

- **Auto Focus**：切换是否在切图时自动聚焦；取消勾选时切图保持 1x
- **Show GT**：显示/隐藏所有 GT 框
- **Show Pred**：显示/隐藏所有 Pred 框

#### Zoom Controls（缩放控制）

- **1x 标签**：显示当前缩放级别
- **[-] 按钮**：缩小一级
- **[+] 按钮**：放大一级
- **Reset 按钮**：重置到 1x 缩放
- **滑块**：拖动调整缩放级别（1x - 10x）

#### Box Actions（框操作）

三个一键操作按钮：

1. **Swap Editable（交换可编辑状态）**
   - 将所有可编辑框变为参考框
   - 将所有参考框变为可编辑框
   - **使用场景**：当 Pred 框质量更好时，交换角色进行编辑

2. **Clear Editable（清除可编辑框）**
   - 删除所有可编辑框
   - **使用场景**：快速删除所有 GT 框，重新标注

3. **Activate Reference（激活参考框）**
   - 将所有参考框变为可编辑框
   - **使用场景**：将 Pred 框合并到 GT，用于补充漏标

#### Save Controls（保存控制）

- **Auto Save**：切换图片时自动保存
  - 开启后，切换图片时自动保存当前修改
  - 配合 Save Unmodified 可保存未改动的 GT

- **Save Unmodified**：未修改也保存当前可编辑框
  - 仅在 Auto Save 开启时生效
  - 适合“确认无误”的样本直接落到 Output

- **Save 按钮**：手动保存当前图片的标注
  - 保存到 Output Path 的临时文件（`{stem}_tmp.txt`）

- **Go Back to Analysis 按钮**：返回 Dashboard
  - 如果有未确认的修改，会弹出确认对话框
  - 选择 Keep：保留修改（临时文件转为正式文件）
  - 选择 Discard：丢弃修改（删除临时文件）
  - 选择 Cancel：取消返回，继续编辑

### 5. 导航操作

- **Prev 按钮**：上一张图片
- **Next 按钮**：下一张图片
- **Header 显示**：当前图片路径、进度（如 15/47）、问题类型、严重程度分数

## 常见操作场景

### 场景 1：修正漏标问题（Overlooked）

1. 在 Dashboard 中只勾选 "Overlooked"，进入标注工具
2. 查看 Pred 框（蓝色虚线），确认是否需要添加到 GT
3. 点击 "Activate Reference" 按钮，将 Pred 框变为可编辑
4. 调整 Pred 框的位置和大小（如果需要）
5. 保存修改

### 场景 2：修正类别错误（Swapped）

1. 在 Dashboard 中只勾选 "Swapped"，进入标注工具
2. 查看问题 GT 框（红色高亮）和对应的 Pred 框
3. 选中 GT 框，使用 `q/w/e/r` 修改类别
4. 保存修改

### 场景 3：修正框位置不准（Bad Located）

1. 在 Dashboard 中只勾选 "Bad Located"，进入标注工具
2. 查看问题 GT 框（红色高亮）和对应的 Pred 框
3. 选中 GT 框，使用鼠标拖拽或键盘方向键调整位置
4. 使用鼠标拖拽控制点或键盘 `a/s/d/f/z/x/c/v` 调整大小
5. 保存修改

### 场景 4：Pred 框质量更好

1. 查看 Pred 框和 GT 框，确认 Pred 框更准确
2. 点击 "Swap Editable" 按钮，交换可编辑状态
3. 现在 Pred 框变为可编辑，GT 框变为参考
4. 编辑 Pred 框（调整位置、大小、类别）
5. 保存修改

### 场景 5：重新标注

1. 点击 "Clear Editable" 按钮，删除所有可编辑框
2. 在空白区域拖拽创建新框
3. 调整框的位置和大小
4. 使用 `q/w/e/r` 设置类别
5. 保存修改

### 场景 6：批量处理

1. 开启 "Auto Save" 开关
2. 如需保存未修改的 GT，开启 "Save Unmodified"
3. 使用 `]` 键快速浏览图片
4. 对每张图片进行必要的修正
5. 系统会在切换图片时自动保存
6. 处理完成后，点击 "Go Back to Analysis"
7. 选择 "Keep" 确认所有修改

2. 启动应用并访问 `http://localhost:8088`

### 测试项目 1：路径配置与验证

**测试步骤**：
1. 在 Dashboard 页面，输入以下路径（其中 GT/Pred/Human 为可选）：
   - Images Path（图片目录，必填）
   - Output Path（输出路径，必填）
   - GT Labels Path（可选）
   - Pred Labels Path（可选）
   - Human Verified Annotation Path（可选）
2. 点击 "Parse Data"
3. 观察每个输入框右侧的状态标签，以及输入框下方的数量提示（valid/missing_img 或 imgs）
3. 尝试输入不存在的路径
4. 尝试将 Output Path 设置为与 GT Labels Path 相同的路径
6. 不填写 GT/Pred 时，确认 Pending Samples 仍会更新（基于已填写的维度）
7. 尝试不设置 Output Path，直接点击 "RUN ANALYSIS"

**预期结果**：
- 路径存在时，状态标签显示绿色 OK，并在输入框下方显示数量
- 路径不存在时，状态标签显示红色和错误信息
- Output Path 与 GT Labels Path 相同时，显示警告但不禁止
- Output Path 未设置时，点击 "RUN ANALYSIS" 会弹窗警告

- [ ] 错误提示是否清晰？
- [ ] 警告信息是否明确？
